    <p>
      -You need an option that provides more than one action.
    </p>
    <p>
      -It is really important that the user stays in context on a phone.
    </p>
    <p>
      -You only have a small number of actions.
    </p>
  </div>
  <div class="wtu">
    <h3>When to use</h3>
    <p>-You need an option that provides more than one action.</p>
    <p>-It is really important that the user stays in context on a phone.</p>
    <p>-You only have a small number of actions.</p>
  </div>
  <div class="wtnu">
    <h3>When not to use</h3>
    <p>-The menu provides only one option. In this case, consider using a button instead.</p>
    <p>-You need to show a hierarchical menu. In this case, use the menu button instead.</p>
    <p>
      -Your users would benefit more from a split button, which offers an easily-accessible default action, with the
      option to include additional actions.
    </p>

    <h2>Content</h2>
    <h3>Anatomy</h3>
    <h5>Title</h5>
    <p>
      -Action sheet title must be clear and all the actions below must be related to the title. The title must be a
      noun, usually the object that the actions are applied upon.
    </p>
    <h5>Actions</h5>
    <p>-Actions must be defined by verbs. For example: “Delete file”</p>
    <p>
      -An action sheet line can include icons, thumbnail pictures. Action text colors can vary according to their
      function. For example a delete action must be red.
    </p>
